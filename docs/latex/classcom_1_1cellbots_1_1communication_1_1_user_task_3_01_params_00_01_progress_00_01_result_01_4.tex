\subsection{Detailed Description}
User\-Task enables proper and easy use of the U\-I thread. This class allows to perform background operations and publish results on the U\-I thread without having to manipulate threads and/or handlers. 

A user task is defined by a computation that runs on a background thread and whose result is published on the U\-I thread. A user task is defined by 3 generic types, called {\ttfamily Params}, {\ttfamily Progress} and {\ttfamily  Result}, and 4 steps, called {\ttfamily begin}, {\ttfamily do\-In\-Background }, {\ttfamily process\-Progress{\ttfamily and {\ttfamily end}. }}

{\ttfamily {\ttfamily  \subsubsection*{Usage}}}

{\ttfamily {\ttfamily  }}

{\ttfamily {\ttfamily  User\-Task must be subclassed to be used. The subclass will override at least one method (\hyperlink{}{do\-In\-Background(\-Object\mbox{[}$\,$\mbox{]})}), and most often will override a second one (\hyperlink{}{on\-Post\-Execute(\-Object)}.) }}

{\ttfamily {\ttfamily  }}

{\ttfamily {\ttfamily  Here is an example of subclassing\-: }}

{\ttfamily {\ttfamily }}

{\ttfamily {\ttfamily 
\begin{DoxyPre}
 private class DownloadFilesTask extends UserTask<URL, Integer, Long> \{ * public File doInBackground(URL... urls) \{ * int count = urls.length; * long totalSize = 0; * for (int i = 0; i < count; i++) \{ * totalSize += Downloader.downloadFile(urls[i]); * publishProgress((int) ((i / (float) count) * 100)); * \} * \} * * public void onProgressUpdate(Integer... progress) \{ * setProgressPercent(progress[0]); * \} * * public void onPostExecute(Long result) \{ * showDialog("Downloaded " + result + " bytes"); * \} * \} *
 \end{DoxyPre}
 }}

{\ttfamily {\ttfamily  Once created, a task is executed very simply\-: }}

{\ttfamily {\ttfamily }}

{\ttfamily {\ttfamily 
\begin{DoxyPre}
 new DownloadFilesTask().execute(new URL[] \{ ... \}); *
 \end{DoxyPre}
}}

{\ttfamily {\ttfamily \subsubsection*{User task's generic types}}}

{\ttfamily {\ttfamily  }}

{\ttfamily {\ttfamily  The three types used by a user task are the following\-: }}

{\ttfamily {\ttfamily  
\begin{DoxyEnumerate}
\item {\ttfamily Params}, the type of the parameters sent to the task upon execution. 
\item {\ttfamily Progress}, the type of the progress units published during the background computation. 
\item {\ttfamily Result}, the type of the result of the background computation. 
\end{DoxyEnumerate}}}

{\ttfamily {\ttfamily  Not all types are always used by a user task. To mark a type as unused, simply use the type \hyperlink{}{Void}\-: }}

{\ttfamily {\ttfamily }}

{\ttfamily {\ttfamily 
\begin{DoxyPre}
 private class MyTask extends UserTask<Void, Void, Void) \{ ... \} *
 \end{DoxyPre}
}}

{\ttfamily {\ttfamily \subsubsection*{The 4 steps}}}

{\ttfamily {\ttfamily  }}

{\ttfamily {\ttfamily  When a user task is executed, the task goes through 4 steps\-: }}

{\ttfamily {\ttfamily  
\begin{DoxyEnumerate}
\item \hyperlink{classcom_1_1cellbots_1_1communication_1_1_user_task_3_01_params_00_01_progress_00_01_result_01_4_a607ca1af80fbed865758fc2d65251484}{on\-Pre\-Execute()}, invoked on the U\-I thread immediately after the task is executed. This step is normally used to setup the task, for instance by showing a progress bar in the user interface. 
\item \hyperlink{}{do\-In\-Background(\-Object\mbox{[}$\,$\mbox{]})}, invoked on the background thread immediately after \hyperlink{}{on\-Pre\-Execute ()} finishes executing. This step is used to perform background computation that can take a long time. The parameters of the user task are passed to this step. The result of the computation must be returned by this step and will be passed back to the last step. This step can also use \hyperlink{}{publish\-Progress(\-Object\mbox{[}$\,$\mbox{]})} to publish one or more units of progress. These values are published on the U\-I thread, in the \hyperlink{}{on\-Progress\-Update(\-Object\mbox{[}$\,$\mbox{]})} step. 
\item \hyperlink{}{on\-Progress\-Update (Object\mbox{[}\mbox{]})}, invoked on the U\-I thread after a call to \hyperlink{}{publish\-Progress(\-Object\mbox{[}$\,$\mbox{]})}. The timing of the execution is undefined. This method is used to display any form of progress in the user interface while the background computation is still executing. For instance, it can be used to animate a progress bar or show logs in a text field. 
\item \hyperlink{}{on\-Post\-Execute (Object)}, invoked on the U\-I thread after the background computation finishes. The result of the background computation is passed to this step as a parameter. 
\end{DoxyEnumerate}\subsubsection*{Threading rules}}}

{\ttfamily {\ttfamily  }}

{\ttfamily {\ttfamily  There are a few threading rules that must be followed for this class to work properly\-: }}

{\ttfamily {\ttfamily  
\begin{DoxyItemize}
\item The task instance must be created on the U\-I thread. 
\item \hyperlink{}{execute(\-Object\mbox{[}$\,$\mbox{]})} must be invoked on the U\-I thread. 
\item Do not call \hyperlink{}{on\-Pre\-Execute ()}, \hyperlink{}{on\-Post\-Execute (Object)}, \hyperlink{}{do\-In\-Background(\-Object\mbox{[}$\,$\mbox{]})}, \hyperlink{}{on\-Progress\-Update (Object\mbox{[}\mbox{]})} manually. 
\item The task can be executed only once (an exception will be thrown if a second execution is attempted.) 
\end{DoxyItemize}}}
